# 组件归一（同 componentId 单定义）实施计划

## 目标

- 在 `dsl:build` 阶段把同一逻辑组件统一到一个定义 ID，避免 `card-38aa4e40` 与 `card-c449b513` 这类并存。
- 不扩展到其它优先项（Tailwind 联动、定位白名单、文本完整 token 化、assets、新约束）以控制改动面。

## 改动范围

- 主改动文件：`[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/build-dsl.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/build-dsl.mjs)`
- 校验补强（可选但建议同批）：`[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/validate.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/validate.mjs)`

## 实施步骤

1. 在 `build-dsl.mjs` 的构建前扫描阶段建立 `componentId 反向索引`：收集所有 `INSTANCE.componentId`，用于识别“`COMPONENT.id` 可映射为同一 componentId”的场景。
2. 抽出 `effectiveComponentId(node)`：
  - 优先 `node.componentId`
  - 否则若 `node.type === "COMPONENT"` 且 `node.id` 出现在反向索引中，使用 `node.id` 作为 componentId
  - 否则返回空（进入现有回退逻辑）
3. 调整 `upsertDefinition()` 的去重 seed：
  - 先走 `cid:${effectiveComponentId}`
  - 无法得到 cid 时再走现有回退（`nid:${node.id}` / hash）
4. 在同一 canonical seed 命中多份候选定义时应用择优规则：
  - 首选 `source.nodeType === "COMPONENT"`
  - 否则保留先到者，仅增加 `usageCount`
5. 保持输出兼容：继续生成 `dsl/definitions/components/*.json` 与 `dsl/registry/components.json`，但同一 componentId 仅出现一个组件定义。
6. 在 `validate.mjs` 增加防回归检查（若同一 componentId 仍映射到多个 definition id 则报错），避免后续逻辑回退导致重复。
7. 本地验证：运行 `npm run dsl:build && npm run dsl:validate`，确认 card 仅保留单定义且 registry 无重复映射。

## 验收标准

- `components.json` 中同一 `source.componentId` 只对应一个 definition id。
- `card-38aa4e40` / `card-c449b513` 仅保留一个（按“优先 COMPONENT”规则）。
- 构建与校验命令均通过。
