## 方案

Hybrid 方案：规则做快速初筛，AI 处理复杂场景。

## 实现

### 1. 新建 Skill

`.cursor/skills/dsl-semantic-inference/SKILL.md`

功能：

- 读取 `dsl/snapshots/structure.latest.json`
- 规则推断 + AI 增强
- 输出 `dsl/snapshots/semantic-skeleton.json` 和 `component-map.json`

### 2. 两阶段推断

**Stage 1: 规则快速推断**（depth ≤ 2）

```js
const QUICK_RULES = [
  { pattern: /header|导航|nav/i, role: "Header" },
  { pattern: /footer|底部|版权/i, role: "Footer" },
  { pattern: /列表|list/i, role: "List" },
  { pattern: /卡片|card/i, role: "Card" },
  { pattern: /按钮|button|提交/i, role: "Button" },
  { pattern: /标题|title|标题/i, role: "Title" },
  { pattern: /图|image|img/i, role: "Image" },
  { pattern: /icon|图标/i, role: "Icon" },
];
// 位置推断
y < 100 && depth === 0 → "Header"
y > 1000 && depth === 0 → "Footer"
```

**Stage 2: AI 增强**（处理规则无法判定的节点）

- 调用 OpenAI API
- 传入未推断的节点 + 上下文

### 3. Prompt

```
分析 DSL 结构，推断语义角色。

AllowedRoles: [Header, Footer, Section, List, Card, Button, Title, Description, Image, Icon, Badge, Nav, SearchBar, ActionButton, Input]

规则：
1. 禁止视觉词：BlueBox/Group1/TopArea
2. depth ≤ 5
3. 结构冻结后不可修改

输入：{structure}

输出 JSON：
{
  "role": "Header",
  "confidence": 0.9,
  "children": [...]
}
```

### 4. 组件抽象检测

- 重复结构 ≥ 3 次 → 抽象组件
- 生成 `component-map.json`

### 5. 环境变量

用户需配置：

- `OPENAI_API_KEY` - OpenAI API Key

## 输出文件

- `dsl/snapshots/semantic-skeleton.json`
- `dsl/snapshots/component-map.json`
