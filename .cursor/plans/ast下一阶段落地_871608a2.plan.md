# AST 下一阶段落地方案

## 目标

把 `docs/ast.md` 末尾“工业级能力”（自动组件拆分、自动 Tailwind、自动 props、自动 import）落到你现有链路 `dsl:build -> dsl:semantic -> dsl:codegen`，不重写 pipeline。

- 语义阶段已输出骨架，但重复检测过于粗糙，`component-map` 目前为空：[/Users/tenghuan/programming/vjshi/code/test-d2c/dsl/snapshots/component-map.json](/Users/tenghuan/programming/vjshi/code/test-d2c/dsl/snapshots/component-map.json)
- 当前代码生成只基于 `role` 生成基础壳组件与模块标题，不是 AST 递归渲染：[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs)
- Tailwind 映射能力已存在并可复用：[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/to-tailwind.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/to-tailwind.mjs)

## 方案（按阶段上线）

### 阶段1：把“自动组件拆分”先做可用

- 改造语义阶段重复检测逻辑，从“同 id 计数”升级为“结构签名聚类 + 最小出现次数阈值”（复用 `extractSignature` 思路，按 role/子结构/ext 归一）。
- 输出稳定的组件候选清单到 `component-map.json`：包含 `name`、`signature`、`fromNodes`、`count`、`role`。
- 文件：[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/semantic-inference.cjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/semantic-inference.cjs)

### 阶段2：升级 codegen 为“AST 递归生成”

- 在 codegen 增加 `renderNode(node)`，按 `semantic-skeleton` 递归产出 JSX，而不是只渲染模块标题。
- 新增组件分层：
  - `app/generated/components/base/*`（role 原子组件）
  - `app/generated/components/composed/*`（由 component-map 抽出的复用组件）
- 自动 import：按实际使用节点收集并去重写入。
- 文件：[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs)、[/Users/tenghuan/programming/vjshi/code/test-d2c/app/generated](/Users/tenghuan/programming/vjshi/code/test-d2c/app/generated)

### 阶段3：接入 Tailwind 与 props 表达式

- 从 `tailwind.latest.json` 读取节点 className 映射，注入到生成 JSX（优先复用已有 mapper 输出，不新造样式系统）。
- props 先做“轻量可用版”：文本节点与图片节点支持表达式占位（如 `{profile.user.username}`、`{hero.image}`），其余节点保持静态。
- 输出统一 `GeneratedPageProps` 类型，放在 `app/generated/page.tsx`。
- 文件：[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs)、[/Users/tenghuan/programming/vjshi/code/test-d2c/dsl/snapshots/tailwind.latest.json](/Users/tenghuan/programming/vjshi/code/test-d2c/dsl/snapshots/tailwind.latest.json)

### 阶段4：命令与验收

- 保持原命令不变，只调整执行顺序建议：`dsl:build -> dsl:semantic -> dsl:tailwind -> dsl:codegen`。
- 验收口径：
  - `component-map.json` 非空且稳定（同输入二次运行差异仅时间戳）。
  - `app/generated/page.tsx` 出现真实 AST 结构渲染（非仅标题循环）。
  - `app/generated/components/composed/` 有拆分产物。
  - `npm run dsl:pipeline` 与 `npm run typecheck` 可通过。

## 实施边界（控成本）

- 不引入新依赖。
- 不改 DSL contract。
- 不做 AI 端到端重写，只在现有 `semantic-inference.cjs` 与 `codegen.mjs` 内增量升级。
