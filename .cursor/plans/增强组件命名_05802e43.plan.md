# 组件命名增强计划

## 目标

- 将当前 `ComposedComponentListeyJyb2xlIjoi184395` 这类名称，改为 `Composed{Module}{Role}{Index}`（如 `ComposedHomeList01`）。
- 保证命名稳定（同一输入多次生成结果一致）、唯一（冲突可自动去重）、可追溯（能看出模块和角色）。

## 改造范围

- 上游命名来源：`[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/semantic-inference.cjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/semantic-inference.cjs)`
- 下游最终命名与落盘：`[/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs](/Users/tenghuan/programming/vjshi/code/test-d2c/scripts/dsl/codegen.mjs)`
- 验证产物：`[/Users/tenghuan/programming/vjshi/code/test-d2c/app/generated/components/composed](/Users/tenghuan/programming/vjshi/code/test-d2c/app/generated/components/composed)`

## 实施步骤

1. 在 `semantic-inference.cjs` 的 `stage4DetectRepetition()` 中，替换 `name: Component_${role}_${seed}` 规则：
  - 从分组信息提取 `moduleId`（清洗为 PascalCase，如 `home_page` -> `HomePage`）。
  - 组合基础名为 `Component_${module}_${role}`，不再拼 base64 片段。
  - 保留 `fromNodes` 排序，确保后续索引稳定。
2. 在 `codegen.mjs` 中重写 `toComposedName()` 命名策略：
  - 输入改为：`module + role + stableIndex`。
  - 输出固定：`Composed${Module}${Role}${Index2}`（两位序号，如 `01`）。
  - 去除对 `fallbackId` 的直接截取拼接（当前随机感来源）。
3. 在 `run()` 里新增命名去重与稳定编号：
  - 先按 `module + role + rootId` 排序生成 `composedDefs`。
  - 对同 `module+role` 组内分配递增序号（`01/02/...`）。
  - 构建 `nodeToComposed` 时使用新名，确保 `emitComposedComponent` 与 `page.tsx` import 全链路一致。
4. 兼容与约束：
  - 名称清洗统一复用现有字母数字规则，首字符非字母时补前缀（避免非法标识符）。
  - 序号不足两位左补零；超过 99 时自然扩展为 3 位，避免截断。
5. 验证与回归：
  - 运行 `npm run dsl:pipeline` 重新生成。
  - 检查 `app/generated/components/composed` 文件名与导出名是否符合 `Composed{Module}{Role}{Index}`。
  - 抽查 `app/generated/page.tsx` 的 composed import 是否全部可解析且无旧命名残留。
